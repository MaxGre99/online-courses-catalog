# Online Courses Catalog

Тестовое задание: карточный каталог курсов с поиском и фильтрацией по категориям.

## Live demo

> Ссылка на задеплоенный проект: [Демо](https://online-courses-catalog-blond.vercel.app/)

## Stack

- **HTML5** -- семантическая разметка
- **SCSS** -- скомпилированный `css/main.css`
- **Vanilla JavaScript** -- без фреймворков и библиотек
- **BEM** -- методология именования классов

## Запуск локально

Просто открыть `index.html` в браузере -- сборка не требуется, CSS уже скомпилирован.

Для разработки (watch-режим SCSS):

```bash
npm install
npm run watch
```

Для пересборки CSS:

```bash
npm run build
```

## Что реализовано

- **Pixel-perfect вёрстка** по макету Figma
- **Fluid / резиновая адаптивность** -- `clamp()` и CSS Grid `auto-fill/minmax` без жёстких брейкпоинтов, плавное масштабирование от 320px до 1920px
- **Живой поиск** по названию и автору курса (debounce 200ms)
- **Фильтрация по категориям**: All / Marketing / Management / HR & Recruting / Design / Development
- **Load more** -- показывается 9 карточек, далее по 3
- **Декоративные элементы** -- точки, шевроны, полукруги из макета
- **Hover-анимации** -- подъём карточки, scale фото, rotate иконки Load more
- **Состояние «нет результатов»** при пустом поиске
- **Доступность** -- `aria-pressed`, `aria-live`, семантические теги, `role="list"`

## Что можно доработать

- Добавить CSS-переходы при смене фильтра (fade-out/in сетки)
- Реализовать lazy-load изображений с IntersectionObserver вместо нативного `loading="lazy"`
- Хранить состояние фильтра/поиска в URL (`?cat=Marketing&q=design`) для шеринга
- Добавить скелетон-плейсхолдеры при загрузке фотографий
- Вынести SCSS в отдельные партиалы (`_variables`, `_catalog`, `_card`) при росте проекта
- Не совсем нравится блок с фильтрами -- по хорошему, или начать прятать их в селектор, или чуть подольше подумать над размещением у каждого брейкпоинта
- Убрать мигание карточек при нажатии на "Load more"
